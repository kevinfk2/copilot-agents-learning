# Agent 基础知识

## 什么是 Agent？

在 GitHub Copilot CLI 中，Agent 是一个自治的 AI 助手，可以：
- 执行特定任务（编码、代码审查、研究等）
- 访问特定的工具集
- 独立做出决策
- 与其他 agents 协作

## 核心概念

### 1. 主 Agent（Main Agent）
- 你正在交互的主要 AI 助手
- 可以委派任务给子代理
- 管理整个工作流程

### 2. 子代理（Subagents）
- 由主 agent 创建来执行特定任务
- 在后台独立运行
- 完成后返回结果

### 3. 内置 Agent 类型

#### Explore Agent（探索代理）
- **用途**：快速查找和分析代码
- **适合场景**：
  - 多个独立的研究问题
  - 大型代码库的交叉分析
  - 并行搜索多个相关区域
- **特点**：只读访问，无法修改文件

#### Task Agent（任务代理）
- **用途**：执行命令和构建任务
- **适合场景**：
  - 运行测试、构建、linting
  - 长时间运行的命令
  - 需要 CLI 工具的任务
- **特点**：返回简洁的成功/失败状态，失败时返回完整输出

#### Code Review Agent（代码审查代理）
- **用途**：分析代码变更
- **适合场景**：
  - 检查 PR 变更
  - 发现 bug 和安全问题
  - 验证最佳实践
- **特点**：高信号、低噪音的反馈

#### General Purpose Agent（通用代理）
- **用途**：复杂的多步骤任务
- **适合场景**：
  - 需要完整工具集的任务
  - 复杂的代码修改
  - 推理密集型工作

## 使用 Agents 的关键命令

### 查看和选择 Agent
```bash
/agent              # 浏览可用的代理
/env                # 查看已加载的所有 agents
```

### 并行执行
```bash
/fleet              # 启用并行子代理模式
/tasks              # 查看所有运行中的任务
```

### 委派任务
```bash
/delegate           # 将任务提交给 GitHub
```

## Agent 选择决策流程

```
任务要求 ↓
├─ 需要代码修改？
│  ├─ YES → 是否需要推理和完整工具集？
│  │         ├─ YES → General Purpose Agent
│  │         └─ NO → 直接处理
│  └─ NO ↓
├─ 只读分析？
│  ├─ YES → 是否是多个独立问题？
│  │         ├─ YES → Explore Agent（可并行）
│  │         └─ NO → grep/view 工具
│  └─ NO ↓
├─ CLI 命令/测试？
│  ├─ YES → Task Agent
│  └─ NO ↓
└─ 代码审查？
   ├─ YES → Code Review Agent
   └─ NO → 直接处理
```

## 何时使用 vs 不使用 Agents

### ✅ 使用 Agents 的情况
- 任务可分解为独立子任务
- 需要后台并行处理
- 任务耗时长（>30秒）
- 任务需要特定的 agent 能力（审查、代码生成等）

### ❌ 不使用 Agents 的情况
- 简单的文件查询（用 grep/view）
- 一行命令修改
- 需要立即反馈的交互式工作
- 单一的简单编辑操作
